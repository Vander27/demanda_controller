# Roteiro Operacional - Preenchimento no Play Console

Atualizado em: 07/05/2026
Aplicativo: Demanda Controller

Objetivo: concluir o preenchimento no Play Console com o menor risco de retrabalho, usando os dados ja validados do aplicativo.

## 1. App content

Entre em: App content.

Preencha nesta ordem:

1. Privacy policy:
- Informar URL: https://vander27.github.io/demanda_controller/privacy-policy.html
- Salvar.

2. Target audience:
- Nao direcionado a criancas.
- Publico profissional/adulto.
- Sem apelo infantil.
- Salvar.

3. News apps:
- Se perguntar se e app de noticias, marcar nao.

4. App access (se aparecer):
- Se nao houver area restrita para revisao, marcar que nao requer acesso especial.

## 2. Data safety

Entre em: App content > Data safety.

Respostas recomendadas:

1. O app coleta dados?
- Sim.

2. O app compartilha dados?
- Sim, apenas com provedores essenciais de infraestrutura.

3. Dados criptografados em transito?
- Sim.

4. Exclusao de dados disponivel?
- Sim.

Categorias para declarar:

1. Informacoes pessoais:
- E-mail.
- Finalidade: funcionalidade do app e gerenciamento de conta.

2. Dados fornecidos pelo usuario/atividade no app:
- Empresas, sites, adiantamentos, lancamentos, relatorios e configuracoes.
- Finalidade: funcionalidade, backup, restauracao e sincronizacao.

Categorias para nao marcar:

- Localizacao.
- Contatos.
- Fotos e videos.
- Audio.
- Saude.
- Biometria.
- Publicidade/rastreamento.

## 3. Store listing

Entre em: Store presence > Main store listing.

Checklist:

1. Categoria:
- Produtividade ou Corporativo (a que melhor refletir sua ficha).

2. Descricao curta e completa:
- Reforcar que e ferramenta profissional de controle operacional e financeiro.

3. URL de politica:
- Repetir a URL da politica oficial no campo correspondente, se existir.

4. Imagens e identidade:
- Garantir que screenshots e icones reflitam telas reais do app.

## 4. Release

Entre em: Release > Production.

Ordem:

1. Criar/editar release.
2. Enviar AAB assinado.
3. Revisar notas da release.
4. Salvar.
5. Revisar e enviar para analise.

## 5. Checklist de envio final

- [ ] Politica de privacidade salva com URL publica.
- [ ] Publico-alvo marcado como adulto/profissional.
- [ ] Data safety preenchido e salvo.
- [ ] AAB anexado na release de producao.
- [ ] Ficha da loja revisada.
- [ ] Envio para revisao realizado.
