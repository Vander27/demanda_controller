Guia operacional para deploy no GitHub Pages

Publicacao da Politica no GitHub Pages

Passo a passo visual para publicar os documentos legais e o guia do app com URL publica valida para o Play Console.

Voltar ao portal Ver politica de privacidade

Objetivo

  1. Publicar a pasta docs no GitHub Pages.
  2. Obter URL publica para preencher no Play Console.
  3. Garantir que politica e termos estejam acessiveis por navegador.
Resultado esperado: pagina inicial do portal funcionando e links legais abrindo sem erro.

Arquivos que devem estar publicados

Portal principal

docs/index.html

Politica de privacidade

docs/privacy-policy.html

Termos e licenca

docs/terms-license.html

Guia de uso do app

docs/guia-usuario-completo.html

Passo a passo no GitHub

  1. Abra o repositorio no GitHub.
  2. Entre em Settings > Pages.
  3. Em Build and deployment, selecione Source: Deploy from a branch.
  4. Escolha Branch: main e Folder: /docs.
  5. Clique em Save e aguarde o deploy.
  6. Atualize a pagina apos 1 a 3 minutos para validar.
Se o site abrir antigo, force refresh no navegador (Ctrl + F5) para limpar cache.

Checklist de conformidade Play Store

Politica de privacidade

  • Nome do aplicativo
  • Data da ultima atualizacao
  • Responsavel e contato oficial
  • Coleta, uso e compartilhamento de dados
  • Armazenamento e seguranca
  • Como excluir conta e dados
  • Nao destinado a criancas

Termos e licenca

  • Aceitacao dos termos
  • Licenca e restricoes de uso
  • Responsabilidades da conta
  • Limite de responsabilidade
  • Propriedade intelectual
  • Suspensao/encerramento
  • Contato oficial
No Play Console, a URL principal obrigatoria geralmente e a da politica de privacidade.
